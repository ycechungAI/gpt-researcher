## 2025-02-28 - [Path Traversal in Report Retrieval]
**Vulnerability:** The `/report/{research_id}` endpoint in `backend/server/app.py` allowed path traversal because it used `os.path.join('outputs', f"{research_id}.docx")` without validating that the resulting absolute path stayed within the `outputs` directory. This could allow an attacker to read any `.docx` file on the filesystem if they could control the `research_id`.
**Learning:** Using `os.path.join` with user input is unsafe because absolute paths or `..` sequences can escape the intended base directory. Even if a framework's router (like FastAPI) provides some implicit protection by rejecting forward slashes in path parameters, relying on this is fragile and can be bypassed on Windows (`\`) or if the route definition changes.
**Prevention:** Always use `os.path.abspath` to resolve the full path and then strictly verify that it resides within the intended directory using `os.path.commonpath([base_dir, target_path]) == base_dir`. Avoid using `startswith` as it is vulnerable to peer-directory traversal bypasses (e.g., `/app/outputs` vs `/app/outputs_secret`).
